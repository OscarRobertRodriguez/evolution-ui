Eyelids

Eyelids is an innovative web component that can be useful to the people who read a lot on the computer screen. It helps you focus on the text or other content in cases of pages with a long chunks of text, or even to help cover up the distracting parts of the complex web page. To use it, just hit Ctrl + 1 on the keyboard. The eyelids will close down vertically on your mouse position and will follow its movement allowing you a clean horizontal area for better focus on the content. To dismiss it, just hit Ctrl + 1 again or Esc key. Optionally, you can add the eyelids notifier on the page with the following code:


  <div class="su_eyelids-notification">
    <p>Use "Eyelids" on this page:</p>
    <code class="su_code"></code>
  </div>

Eyelids has the capability to adjust the width of the highlighted area on the screen. To do that, press "Shift" key and scroll with your mousewheel.

Modals

There are three modal types, simple, image and video modal.

Simple modal component provides clean and quick way to display simple textual information to the user. Modals with image and video provide additional media content besides the textual info.

On the page you need only one modal container. The code structure for the modal container is simple:


  <div class="su_modal">
    <span class="su_modal-close material-icons">close</span>
    <div class="su_modal-content su_align-center">
      <h4 class="su_modal-title"></h4>
      <div class="su_modal-media"></div>
    </div>
  </div>

To display the information in a modal, a toggle element is needed with the data-modal-trigger attribute set. This attribute can have the following values: "simple" - for presenting a simple textual information, "image" - for displaying an image and "video" - for displaying a video content (YouTube video embed supported only).

Modal trigger element can be any element with appropriate "data" attributes:

A simple modal:

<span data-modal-type="simple" data-modal-title="Nothing to see here, please move along!" data-modal-media="This is the simple modal that present some text content to the user. This content comes from the 'data-modal-media' attribute set on the element that triggers the modal.">Click to learn more</span>

The data-modal-title attribute holds the title to be displayed on the modal, while the data-modal-media holds the media that is to be displayed in the modal. In the case of the simple modal, that is plain text.

In the case of the image and video modal, the data-modal-media attribute should provide the link to the media resource. In addition, if the image is used as the trigger for the image modal, and the link is not provided through data-modal-media attribute, then the modal will display the image from the modal trigger, by using its src attribute.

Dot Navigation

Using dots as the navigation through panels, tabs, and image carousels is a common practice on the web. But do those dots need to be common? Why should they? They can be interesting component by themselves, and in that way improve the user experience and delight while using them. Here are some interesting effects for dot navigation.

Burst

Scale

Jiggle

Zap

The code for dot navigation is very simple. You need a container, an unordered list, and three classes. Basic classes are su_dot-navigation on the container and su_dot-current on the currently active dot. You can add su_dot-current class in the markup on the dot that you want initially to be active. While using the navigation, this class will be dynamically removed and added to the selected dot, thus triggering the animation effect. Third important class is a modifier class that determines the animation effect. There are four classes that determine the effect:

  • su_dot-navigation-burst

Notice that the list items do not have any content. Do not add anything in the list items. (I think that I should safeguard from this case by removing any content that the user might have enterd on the list items.)

The dot navigation container has its display set to "inline-block" which makes it easy to align using the "text-align" CSS rules. So far there are only these four effects, but in the future there might be much more of them.

Off Canvas Navigation

Off canvas navigation is an nice way of getting the navigation menu out of the way so that the page content has more space to be displayed, while also providing a nice effects of accessing the menu and while the menu is displayed it has primary focus on the page.

To use the off canvas navigation component, the markup of the page must follow a specific structure. Here is how that structure looks like:


  <!-- the off canvas menu -->
  <div class="su_offcanvas-menu-slide-in">
    <aside class="su_aside su_left">
      <nav>
        <ul>
          <li><a href="#">Link 1</a></li>
          <li><a href="#">Link 2<a></li>
          <li class="su_sidenav-current"><a href="#">Link 3</a></li>
          <li><a href="#">Long Link 4</a></li>
          <li><a href="#">A Really Really Long Link 5</a></li>
          <li><a href="#">Link 6</a></li>
        </ul>
      </nav>
    </aside>
  </div>

  <!-- wrapper for the page content -->
  <div class="su_offcanvas-wrapper">
    <div>
      <!-- page content -->
    </div>
  </div>

Notice that this markup uses the side navigation bar markup for navigation menu.

Triggering the off canvas navigation menu

To trigger the off canvas menu, you have to add su_menu-triggers class to the trigger element, most likely a "button" element, and you must provide the data-menu-effect attribute to define the revealing effect. Here is an example button:

  <button class="su_menu-triggers su_btn su_btn-primary" data-menu-effect="slide-in">Slide in</button>

Notice that the data-menu-effect must correspond to the class name of the menu su_offcanvas-menu-slide-in for the desired revealing effect to work. Here is the table of the revealing effects and corresponding classes:

Revealing Effect "data-menu-effect" value Class applied to the menu
Slide In slide-in su_offcanvas-menu-slide-in
Slide Over slide-over su_offcanvas-menu-slide-over
Reveal reveal su_offcanvas-menu-reveal

Classes su_aside and su_left are also important to be applied to the <aside> element. At the moment, it is not possible to use the class su_right to place the off canvas navigation to the right. Optionally, you can apply class su_theme-light to create a light colored menu.
